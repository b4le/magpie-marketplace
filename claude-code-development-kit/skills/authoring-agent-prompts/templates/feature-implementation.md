# Feature Implementation Template

## Purpose
A structured approach to implementing new features with clear requirements, constraints, and success criteria.

## Usage
Use this template when requesting implementation of new features to ensure all aspects are considered and tracked.

**Best for:**
- New user-facing features
- API endpoint development
- Component creation
- Module additions

**Tip:** Combine with `TaskCreate`/`TaskUpdate` for progress tracking on multi-file features.

## Template

```
Implement [feature name].

Context:
[Why this feature is needed, user story, business value]

Requirements:
1. [Specific requirement]
2. [Specific requirement]
3. [Specific requirement]

Technical Constraints:
- Use [technology/pattern]
- Follow [convention]
- Integrate with [existing system]

Success Criteria:
- [ ] All requirements met
- [ ] Tests written and passing
- [ ] Documentation updated
- [ ] No new linting errors
- [ ] Backwards compatible (if applicable)

Use TaskCreate/TaskUpdate to track progress.
```

# Documentation Template

## Purpose
A comprehensive guide for creating effective documentation with clear audience, structure, and practical examples.

## Usage
Use this template when requesting documentation creation to ensure all necessary sections are included and examples are practical.

**Best for:**
- API documentation
- README files
- User guides
- Technical specifications

**Best Practice:** Include runnable examples that users can copy-paste directly.

## Template

```
Create documentation for [feature/API/component].

Audience: [who will read this]

Include:
- Purpose and overview
- Getting started / quick start
- Detailed usage examples
- API reference (if applicable)
- Configuration options
- Common issues and solutions
- Related resources

Format: [Markdown/JSDoc/OpenAPI/etc.]

Examples should be:
- Runnable (where possible)
- Cover common use cases
- Include error handling
```

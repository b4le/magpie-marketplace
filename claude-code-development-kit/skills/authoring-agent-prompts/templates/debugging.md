# Debugging Template

## Purpose
A systematic approach to debugging issues with clear reproduction steps, expected vs actual behavior, and investigation methodology.

## Usage
Use this template when reporting bugs or requesting debugging assistance to ensure thorough investigation and resolution.

**Best for:**
- Production bugs
- Test failures
- Unexpected behavior
- Performance issues

**Tip:** Include error messages, stack traces, and relevant logs for faster diagnosis.

## Template

```
Debug [issue description].

Steps to reproduce:
1. [Step]
2. [Step]
3. [Step]

Expected behavior:
[What should happen]

Actual behavior:
[What actually happens]

Investigation approach:
1. Examine relevant code
2. Check logs for errors
3. Review recent changes
4. Test hypotheses
5. Implement fix
6. Verify fix works
7. Add tests to prevent regression

Explain your findings and reasoning.
```

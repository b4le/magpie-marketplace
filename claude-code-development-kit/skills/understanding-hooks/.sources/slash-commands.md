# Hooks and Slash Commands

## The /hooks Command

The `/hooks` command is a built-in slash command for managing hook configurations for tool events.

## Usage
Use `/hooks` to:
- Configure hooks for different events
- Manage hook settings interactively
- Set up matchers and commands

## Minimal Integration
While hooks can be configured to respond to various Claude Code events, there is limited direct interaction between hooks and slash commands documented. The primary connection is through the `/hooks` command itself for configuration.

## Reference
For more detailed information about hooks, refer to the IAM (Identity and Access Management) guide or the CLI reference documentation.
